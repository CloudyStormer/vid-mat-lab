# Codex 工作约定

本仓库用于持续制作“内耗儿绝缘体”系列短视频。任何新会话开始工作前，先完整读取：

1. README.md
2. docs/ACCOUNT_POSITIONING.md
3. docs/VIDEO_WORKFLOW.md
4. episodes/README.md
5. 当前要处理的单集 README.md

## 不依赖聊天上下文

- 仓库文档是唯一长期上下文。
- 重要决定、最终文案、素材路径、剪辑参数、发布状态和复盘必须写回对应单集。
- 不要只在聊天中交付关键内容。

## 单集结构

- episodes 下面每个编号子目录代表一集正式视频。
- 新一集使用三位编号加英文短标题，例如 002-workplace-emotional-labor。
- 从 templates/episode 复制结构，不覆盖旧集。
- 历史版本要标注 v1、v2 或 archive，不要把未确认版本伪装成最终版。

## 内容原则

- 账号主线是年轻打工人的工作、生活、金钱和关系压力，以及减少内耗。
- 健身、吃饭、通勤和兴趣爱好属于轻入口，不应把账号变成单一健身号。
- 借公共话题和事实，不搬运他人的成片、文案、配音、镜头组合或独特表达。
- 健康、法律、财务等内容要核验可靠来源，使用有条件的表述。
- 语言自然、干练、专业，但不要像论文、公告或说教。
- 结尾尽量回到“压力”和“减少内耗”，不要生硬喊口号。

## 视觉原则

- 固定 IP 参考图在 assets/brand/resistor-mascot-reference.png。
- 必须保留白色圆形电阻身体、灰色电阻腰带、锯齿电阻纹、蓝色电路纹、黄色闪电和黑色粗线条。
- 当前主视频为横版 16:9；图片默认无文字，文字在剪辑或发布阶段添加。
- 封面应预留清晰标题区，避免复杂背景和 PPT 式排版。

## 交付与 Git

- 修改前检查 git status，保留用户已有改动。
- 每次工作结束前更新单集 README 和 episodes/README.md。
- 在合理范围内验证 Markdown 链接、文件存在性和 Git diff。
- 用户要求推送时，完成提交并推送当前分支；只有推送成功才报告完成。
