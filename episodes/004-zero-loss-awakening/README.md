## 本集目标

- 面向谁：刚开始做内容、容易被低播放量影响情绪的人。
- 解决什么问题：不做说教，用一个五秒冷幽默短梗释放“新号没流量”的创作内耗。
- 观众看完得到什么：一个能迅速理解并可能愿意转发的情绪笑点。
- 与账号主线的关系：属于“创作内耗”轻短梗，并用科技觉醒形态强化固定 IP。

## 核心创意

科技形态的小电阻人检查一颗微型星球，冷静得出“没有流量”的结论，然后把它当作失败的虚拟模拟一样轻轻删除。笑点来自离谱行动与平静表情的反差，不表现真实灾难。

## 成片结构

1. 角色托着微型星球扫描，弱信号闪烁。
2. 画外音：“不行，这颗星球没流量。”
3. 角色轻点星球，画外音：“炸了算了。”
4. 星球“噗”成蓝白数据云；角色面无表情地停顿半秒。

## 遇到的问题与解决办法

| 问题 | 原因 | 解决办法 | 下次预防 |
|---|---|---|---|
| AI 容易把灰色腰带和锯齿纹生成第二张嘴 | 角色腹部图案具有牙齿般的视觉结构 | 提示词明确锁定唯一嘴部位置；视频不做自动口型，后期加画外音 | 每个视频提示词复用“单嘴、腰带不是嘴”的硬约束 |
| 单纯展示变身缺少具体笑点 | 只有视觉炫技，没有与账号处境连接 | 改为“没流量就换颗星球”的五秒创作内耗短梗 | 先写一句可复述的笑点，再决定视觉特效 |
| 真实爆炸会破坏轻松感 | “炸掉星球”容易被模型理解成灾难画面 | 将爆炸定义为虚拟星球变成蓝白数据云的轻巧“噗” | 明确禁止火球、浓烟、城市和灾难镜头 |

## 下一步

1. 在即梦或可灵使用首尾帧模式生成五秒动态。
2. 在剪映加入画外音、扫描音、轻触音和小型数据“噗”音。
3. 检查唯一嘴部、手指数量、星球变化和 16:9 画幅后导出。
4. 发布后记录 24/72 小时的五秒留存、完播和转发数据。
