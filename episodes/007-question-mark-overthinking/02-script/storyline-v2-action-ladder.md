# 故事线 v2｜一个问号后的四步行动

## 本版调整

- 删除上一版对“脑补”的重复解释。
- 增加第一次澄清、未回复后的追问、紧急事项换渠道、到期仍未回复的处理。
- 增加关键事项不能把沉默当同意的边界。
- 目标仍为约一分钟说明型口播。

## 精简口播

```text
同事只回一个问号，你会不会立刻脑补：不耐烦？质疑我？还是准备甩锅？

先停一下。事实只有一个：他没看懂，或者需要确认；其他态度，都还没有证据。

先直接问：你是没看懂哪一部分，还是对结论有疑问？

如果没回，就把背景、要他确认的事和时间点说清楚：这个决定会影响今天的排期，麻烦三点前确认。

事情紧急，就打电话或者当面说；不紧急，就给对方合理的回复时间。

到点还没回，可以留痕：我先按 A 方案推进，有调整请告诉我。

但付款、发布、合同这些需要明确授权的事，不能把沉默当成同意，要继续确认或者找负责人。

职场沟通别靠猜：问题问具体，期限说清楚，结果留记录。

一个问号，不值得你脑补一整晚。
```

## 时长预估

- 约 230 个汉字。
- 正常解释语速并保留短暂停顿，预计 55—62 秒。

## 口播中的三条可复制话术

### 第一次澄清

```text
你是没看懂哪一部分，还是对结论有疑问？
```

### 带时间点追问

```text
这个决定会影响今天的排期，麻烦三点前确认。
```

### 普通事项到期留痕

```text
为了不影响排期，我先按 A 方案推进；如果需要调整，请告诉我。
```
