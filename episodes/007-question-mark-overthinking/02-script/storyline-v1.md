# 故事线 v1｜同事只回了一个问号

## 一句话故事

小电阻人发出一段工作消息，同事只回了一个问号；它在脑中演完一整部职场悬疑片，最后才发现，与其猜测对方的态度，不如直接问清哪里没有说明白。

## 一分钟故事线

### 1. 钩子：一个问号

小电阻人刚把工作说明发给同事，聊天框里很快弹出一个孤零零的“？”。

它盯着屏幕，身体瞬间僵住。

### 2. 脑内会议开始

小电阻人的脑中同时出现几种声音：

- 他是不是不耐烦了？
- 是不是觉得我表达得很差？
- 他是在质疑我，还是准备把事情甩回来？

现实里只有一个问号，脑子里却已经吵了三轮。

### 3. 内耗扩大

小电阻人开始反复检查自己刚才的消息，删除又重写，甚至后悔为什么要主动沟通。

一个普通符号，被它扩写成了“别人正在否定我”。

### 4. 暂停：事实是什么

画面停住，把信息分成两边：

- 已知事实：对方发了一个问号。
- 未经确认：不耐烦、否定、针对、甩锅。

这些解释可能成立，也可能完全不成立。

### 5. 真正的问题

对方可能只是没看懂某句话、不知道附件在哪里，或者想确认任务的具体范围。

在信息不足时，大脑会自动补齐原因，但补出来的不等于事实。

### 6. 换一个动作

小电阻人不再反复修改，也不替对方预设态度，直接回复：

“是哪个地方没说清楚？我补充一下。”

### 7. 沟通恢复正常

对方指出具体问题，小电阻人补上缺失信息。事情很快继续推进，并没有发生它脑补的冲突。

这里不强调“对方一定没有恶意”，只强调：先确认，再判断。

### 8. 收尾

对方发来的只是一个问号，别让自己的脑子把它扩写成一整部职场悬疑片。

少猜一点，多问一句，内耗就少一半。

## 连贯口播初稿

```text
你有没有遇到过这种情况？

给同事发了一段工作消息，对方只回了一个问号。

你盯着屏幕，脑子马上开会：他是不耐烦？觉得我没说清楚？还是在质疑我？

明明只有一个符号，你却已经在心里吵了三轮，甚至开始后悔刚才那段话。

可事实是，对方只发了一个问号。至于不耐烦、否定和针对，都是我们在信息不完整时补上的剧情。

他可能只是没看懂一句话，也可能只是没找到附件。

遇到这种情况，先别替对方写台词，也别反复删改自己的消息。

直接问：是哪个地方没说清楚？我补充一下。

能确认的就确认，能澄清的就澄清。

别人发来的只是一个问号，别让自己的脑子，把它扩写成一整部职场悬疑片。

少猜一点，多问一句，内耗就少一半。
```

## 时长预估

- 正常、略带停顿的中文口播：约 55—65 秒。
- 若最终配音超过 65 秒，优先删去“能确认的就确认，能澄清的就澄清”，不建议整体加速。

## 表达边界

- 不把“敏感”说成性格缺陷。
- 不武断判断同事没有恶意。
- 不鼓励观众忽略真实的冒犯或职场风险。
- 本集只解决一个动作：信息不足时先澄清，再决定如何回应。
